Log Analysis Home Lab — Splunk Enterprise

Splunk Enterprise · Ubuntu (VM) · Zeek Logs · SPL · Investigation Workflows

Overview

Built a SIEM-style log analysis lab using Splunk Enterprise on an Ubuntu virtual machine, accessed from a Windows host. The objective was to ingest real security telemetry, write SPL queries, and practise investigation workflows aligned to SOC tasks.

Lab Setup

  • Installed and configured Splunk Enterprise on Ubuntu (VM) with service persistence and Splunk Web access.
  • Prepared structured log sources for ingestion (Zeek-style logs for network activity and authentication-related events).
  • Configured sourcetypes/indexing to support consistent searching and analysis.

What I Did

  • Ingested Zeek DNS, SSH, HTTP, and connection logs into Splunk for analysis.
  • Developed SPL queries to baseline behaviour and identify anomalies across network and authentication events.
  • Analysed DNS activity to identify frequently queried domains, top clients, and query types.
  • Investigated SSH activity to identify failed authentication spikes and high-risk sources.
  • Reviewed HTTP/connection activity to understand traffic patterns and identify unusual behaviour.

What I Tested and Validated

  • Baseline vs anomaly: compared normal traffic patterns against suspicious spikes and outliers.
  • Auth visibility: confirmed detection of repeated failed SSH attempts and abnormal connection behaviour.
  • Query confidence: validated SPL outputs by cross-checking counts, time ranges, and fields.

Key Outcomes

  • Improved practical SOC skills: log ingestion, SPL searching, and investigation structure.
  • Built repeatable dashboards/queries for DNS and SSH activity reviews.
  • Strengthened understanding of how SIEM tooling supports detection and triage workflows.

Tools

  • Splunk Enterprise
  • Ubuntu (Virtual Machine) + Windows host access
  • Zeek logs (DNS / SSH / HTTP / conn)
  • SPL (Search Processing Language)
