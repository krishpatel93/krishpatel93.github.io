Network Firewall, Traffic Analysis & DoS Mitigation — pfSense

pfSense · DoS Simulation · Firewall Rules · Logging · Wireshark Analysis

Overview

Built a network security lab to understand denial-of-service behaviour, firewall tuning, and network visibility. The objective was to simulate attack traffic, observe patterns in packet captures, and implement mitigations using pfSense rules and logging.

Lab Setup

  • Designed a segmented lab network with WAN/LAN interfaces and a protected internal host.
  • Configured pfSense as the perimeter firewall and gateway for the environment.
  • Prepared tooling for packet capture and analysis to validate behaviour before and after mitigations.

What I Did

  • Generated controlled DoS-style traffic to reproduce symptoms such as increased latency and packet volume spikes.
  • Captured and analysed traffic to identify patterns consistent with flood behaviour.
  • Implemented firewall rules and rate-limiting style controls to reduce malicious traffic impact.
  • Monitored firewall logs and interface statistics to confirm mitigation effectiveness.

What I Tested and Validated

  • Traffic visibility: validated attack signatures using packet captures and log evidence.
  • Rule effectiveness: confirmed firewall controls reduced or blocked attack traffic reaching the internal host.
  • Before vs after: compared baseline traffic behaviour against attack conditions and mitigated conditions.

Key Outcomes

  • Strengthened practical understanding of firewall operation, logging, and traffic-based troubleshooting.
  • Built confidence using packet captures to explain what happened on the network and why.
  • Developed a repeatable approach: simulate → observe → control → validate.

Tools

  • pfSense
  • Wireshark
  • Attack simulation tooling (e.g., ICMP/SYN flood generators)
  • Virtualised lab environment
