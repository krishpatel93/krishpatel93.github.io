AWS S3 Secure Storage & Access Logging

Amazon S3 · Access Control · Server Access Logging · Cloud Security Fundamentals

Overview

This project focused on securing cloud object storage using Amazon S3 and enabling detailed access logging to support visibility, auditing, and traceability. The objective was to understand how storage misconfiguration can introduce risk and how AWS-native controls can be used to prevent unauthorised access.

Objectives

  • Configure secure S3 buckets following least-privilege principles.
  • Prevent public access to sensitive storage resources.
  • Enable and validate server access logging.
  • Understand how storage-level security contributes to cloud defence-in-depth.

Architecture Overview (Conceptual)

  • Primary S3 bucket configured for secure object storage.
  • Dedicated log bucket receiving S3 server access logs.
  • AWS-managed permissions enforcing controlled access.
  • No public access permitted at the bucket or object level.

What I Did

  • Created and configured an S3 bucket with public access fully blocked.
  • Reviewed and validated bucket permissions to ensure no unintended exposure.
  • Enabled S3 server access logging to capture detailed request-level activity.
  • Configured a separate destination bucket to securely store access logs.
  • Performed controlled object uploads and access attempts to generate log entries.

Validation & Testing

  • Confirmed objects could be uploaded and accessed only through authorised actions.
  • Verified that public access was blocked at both bucket and object levels.
  • Validated that access logs were generated and stored in the logging bucket.
  • Reviewed log entries to understand request source, time, and action details.

Final Security Posture

  • Access control: public access disabled, permissions restricted.
  • Visibility: detailed server access logs enabled and retained.
  • Separation of duties: logging isolated in a dedicated bucket.

Key Learnings

  • Hands-on understanding of S3 security controls and common misconfiguration risks.
  • Practical experience enabling and validating cloud-native logging.
  • Improved awareness of how storage security supports audit and incident response.

Possible Improvements

  • Integrate S3 access logs with a SIEM for centralised monitoring.
  • Apply lifecycle policies to manage log retention and cost.
  • Encrypt objects and logs using customer-managed KMS keys.
  • Implement automated compliance checks using AWS Config.
