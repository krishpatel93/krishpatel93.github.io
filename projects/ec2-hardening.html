Securing an AWS EC2 Instance — Cloud Security Fundamentals

AWS EC2 · Amazon Linux 2023 · Security Groups · SSH Key Authentication · Linux Hardening

Overview

This project focused on deploying and securing an internet-facing virtual server in Amazon Web Services (AWS). The objective was to understand how cloud networking and operating system hardening work together to reduce attack surface while maintaining secure administrative access.

Objectives

  • Deploy a Linux-based virtual server in AWS.
  • Secure remote access using industry-standard authentication methods.
  • Restrict network access to trusted sources only.
  • Harden the server against common attack vectors.
  • Understand the interaction between AWS networking and host-level security.

Architecture Overview (Conceptual)

  • Single EC2 instance running Amazon Linux 2023.
  • AWS Security Group acting as a network firewall.
  • SSH key-based authentication.
  • Host-level SSH hardening on the Linux server.
  • Access restricted to a single trusted public IP and non-root user.

Implementation Summary

  • Deployed the EC2 instance in the ap-southeast-2 (Sydney) region using an IAM user.
  • Generated and configured an SSH key pair for secure authentication.
  • Configured security group rules to allow SSH access only from a trusted IP address.
  • Updated the operating system to apply the latest security patches.
  • Hardened SSH by disabling root login and password-based authentication.
  • Restarted and validated SSH service functionality after configuration changes.

Validation & Testing

  • Verified SSH access using key-based authentication.
  • Confirmed password authentication was blocked.
  • Validated that root SSH access was disabled.
  • Confirmed network access was restricted correctly by the security group.

Final Security Posture

  • Network layer: SSH restricted to a single trusted IP.
  • Authentication layer: SSH keys enforced, passwords disabled.
  • Privilege layer: Root login disabled; access via non-root user.

Key Learnings

  • Hands-on experience deploying and securing cloud-based servers.
  • Practical understanding of AWS security groups as network firewalls.
  • Real-world Linux hardening techniques for internet-facing systems.
  • Defense-in-depth using both cloud-native and host-based controls.

Possible Improvements

  • Integrate CloudWatch monitoring and alerts.
  • Use IAM roles instead of static credentials.
  • Assign Elastic IPs for stable addressing.
  • Automate builds using infrastructure-as-code tools.
  • Extend security controls to AWS S3 storage.
